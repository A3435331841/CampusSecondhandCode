# 肖家鑫 - 第一阶段（需求分析与架构搭建）工作小结

## 1. 本阶段工作内容
*   **后端技术选型与骨架搭建**：
    *   基于 **Spring Boot 3.2.3** 搭建了高性能后端架构，集成了 **MyBatis-Plus** 作为持久层框架，**Redis** 作为缓存中间件。
    *   完成了统一异常处理、全局结果封装类 `Result.java` 的编写，规范了 API 输出格式。
*   **数据库建模与持久化**：
    *   使用 MySQL 8.0 设计了核心业务表结构，包括用户表（`sys_user`）、商品表（`biz_product`）、订单表（`biz_order`），并完成了 SQL 初始化脚本 `database.sql`。
*   **高并发核心业务开发**：
    *   在 `OrderServiceImpl.java` 中实现了基于 **Redis Lua 脚本** 的原子性库存预扣减逻辑，确保在秒杀/高频交易场景下系统不超卖。
*   **安全认证体系**：
    *   设计并实现了基于 **JWT/Token** 的用户登录鉴权流程，保障了系统接口访问的安全性。

## 2. 遇到的问题以及解决思路
*   **问题 A：高并发下的库存超卖问题**
    *   **现象**：在初始测试中，由于数据库 `UPDATE` 语句在并发请求下存在行锁竞争和数据不一致，导致实际卖出的商品数量超过了库存。
    *   **解决思路**：引入 Redis 缓存库存。通过编写 **Lua 脚本** 将“查询库存”和“扣减库存”封装为一条原子指令在 Redis 中执行。
    *   **优化**：针对 Redis 缓存失效的情况（缓存击穿），实现了双重检查锁定逻辑，从数据库自动加载库存并同步至 Redis。
*   **问题 B：Spring Boot 3.x 版本兼容性问题**
    *   **现象**：由于 Spring Boot 3.x 弃用了某些旧版依赖（如 Java EE 迁移到 Jakarta EE），导致部分插件报错。
    *   **解决思路**：查阅官方迁移指南，更新了所有核心依赖版本，并手动调整了 `javax.*` 到 `jakarta.*` 的包引用。

## 3. 下一阶段计划
